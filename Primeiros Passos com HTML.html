Entendendo Comunicação Cliente x Server

Voltar

Para entender como e dá a comunicação entre cliente e server, devemos entender o que é cada parte:

  1. Server:
    2. São locais físicos em que há uma rede de dispositivos conectados, que podem acessar arquivos e informações nele contidos, ou servidores de internet, em que as informações que os Clientes terão acesso estão hospedadas.

    Existem tipos diversos de servidores, que se correlacionam à segurança (Firewall), Web, e-mail, Proxy (cria um filtro numa rede interna), etc.

  2. Cliente(ou Client):
    3. É o dispositivo que vai consumir a informação que está contida no server.

Assim, a comunicação entre Cliente e Server ocorre através da internet; o server serve informações para os clients, eles as consomem e também devolvem informações para o server.

Introdução ao HTML na prática

Voltar

  • Página principal: Convencionalmente, possui o nome de index.html.

  • O HTML consiste em 3 partes:
    1. Tag HTML: É o elemento raíz, definindo todo o documento HTML, ou seja, tudo o que estiver entre a tag de abertura e fechamento é um documento HTML. O navegador HTML consiste nas outras duas partes:
    2. Tag HEAD: Infrmações pré carregadas antes de serem exibidas ao usuário.
    3. Tag BODY: O que irá ser exibido ao usuário.

  • Tags: Existem milhares, e cada uma possui uma função. A informação inserida entre as tags de abertura e fechamento será interpretada pelo navegador da forma definida por ela. Algumas tags, no entanto, não possuem fechamento.

    • As tags podem conter atributos, que são propriedades que elas podem ter. Podem ser globais, ou seja, praticamente qualquer tag pode ter, ou específicos, que são designados apenas para uma tag específica.

  • Texto: É basicamente o que será lido pelo usuário. Possui inúmeras tags específicas para alterar sua estrutura e seu layout, desde tamanhos de título e subtítulo (em até 6 níveis), definir parágrafos e até incluir citação.

  • Listas: Podem ser de duas formas (explicadas abaixo), definidas pelas tags OL ou UL, e seus itens devem estar aninhados e entre as tags de abertura e fechamento li. Pode-se também inserir uma lista aninhada em uma lista.

    • Lista ordenada OL: Aparece em ordem numérica.
    • Lista não ordenada UL: Aparece como marcadores ao invés de números. O padrão são marcadores como pontos, mas isso pode ser alterado.

  • Links: Juntam um elemento em outro, através da tag de ancoragem a. Assim, direcionam o usuário a partes da própria página, outras páginas do proprio site ou a sites terceiros.
    Possui atributos que podem abrir outras páginas na mesma guia, atualizando-a (interessante quando são páginas do mesmo site, para manter o usuário dentro dele) ou em outra guia (geralmente usado para abir páginas externas ao site).
